# Twitch Chat Viewer Bot

**Twitch Chat Viewer Bot** — это мощное десктопное приложение для управления множеством Twitch-ботов одновременно. Оно предоставляет удобный пользовательский интерфейс для оркестрации ботов, автоматического взаимодействия в чате (включая AI-генерацию сообщений) и эмуляции реальных зрителей на стриме.

## 🌟 Ключевые особенности

- **Масштабируемое управление ботами**: Параллельное подключение множества ботов с учетом лимитов (Rate Limiting), поддержкой автоматического переподключения (Reconnector) и обработкой банов/таймаутов.
- **Интерактивный дашборд (React)**: Живая статистика, графики онлайна (Chart.js), предпросмотр чата в реальном времени и удобная таблица аккаунтов с фильтрацией и поиском.
- **Умный чат-движок**: Отслеживание контекста бесед (ContextTracker), вероятностные ответы на упоминания, диалоги между ботами и генерация сообщений через фиксированные списки или OpenAI API с кэшированием.
- **Эмуляция зрителей (Viewer Service)**: Использование Puppeteer (с плагинами stealth и ghost-cursor) для надежного запуска трансляций и имитации активности реальных пользователей.
- **Логирование и аналитика**: Сохранение истории сообщений в локальную базу данных SQLite с циклическим буфером и автоматической очисткой старых записей.
- **Удобный импорт**: Поддержка массового добавления аккаунтов через текстовые файлы (Drag-and-Drop) или прямое вставление (Bulk Paste) с валидацией.

## 🛠 Технологический стек

- **Desktop Framework**: Electron
- **Frontend**: React, Zustand (стейт-менеджмент), Tailwind CSS
- **Backend/Core**: Node.js, TypeScript
- **Взаимодействие с Twitch**: tmi.js
- **Эмуляция браузера**: Puppeteer, puppeteer-extra (stealth plugin)
- **База данных**: SQLite (better-sqlite3)
- **AI интеграция**: OpenAI API

## 💻 Системные требования

- **ОС**: Windows, macOS или Linux
- **Node.js**: Версия `>=20.0.0`
- **Пакетный менеджер**: `npm` или `yarn`
- ⚠️ **Важно**: Для стабильной работы модуля эмуляции зрителей (Viewer Service) настоятельно рекомендуется использовать **индивидуальные серверные (Datacenter) IPv4 прокси**. Использование публичных или забаненных IP может привести к капчам или блокировке со стороны Twitch.

## 🚀 Установка и запуск

1. **Клонируйте репозиторий**:
   ```bash
   git clone https://github.com/w436212-beep/Twitch-bots.git
   cd Twitch-bots
   ```

2. **Установите зависимости** (рекомендуется использовать флаг `--prefer-offline` для стабильности):
   ```bash
   npm install --prefer-offline
   ```

3. **Запуск в режиме разработки**:
   ```bash
   npm run dev
   ```
   Эта команда скомпилирует основной процесс и UI, после чего запустит приложение Electron.

## ⚙️ Конфигурация

Приложение гибко настраивается через конфигурационные файлы и интерфейс:

- **`src/config/defaults.json`**: Содержит базовые настройки приложения (лимиты, тайминги, вероятности ответов и т.д.). Изменения настроек через UI безопасно сохраняются на диск.
- **Переменные окружения (`.env`)**: Создайте файл `.env` в корне проекта для хранения чувствительных данных (например, ключей OpenAI API). Токены автоматически маскируются в логах приложения для безопасности.
- **Формат аккаунтов**: Список ботов можно импортировать через текстовые файлы. Каждая строка должна содержать данные аккаунта (включая oauth токен). Поддерживается импорт напрямую через интерфейс (кнопка Bulk Paste или Drag-and-Drop).

## 📖 Использование (Quick Start)

1. **Добавление аккаунтов**: Откройте дашборд и перетащите текстовый файл с аккаунтами в область таблицы или используйте поле для массовой вставки (Bulk Paste). Приложение автоматически проверит формат и валидность аккаунтов.
2. **Настройка канала**: В панели управления (Controls Panel) введите имя целевого Twitch-канала.
3. **Выбор режима чата**: Выберите между фиксированными фразами (Fixed) или генерацией через ИИ (AI Mode). При выборе AI убедитесь, что ключ API настроен в настройках (Settings).
4. **Запуск ботов**: Нажмите кнопку **Start**, чтобы инициировать параллельное подключение. Боты автоматически распределят задержки (jitter) для безопасного входа в чат.
5. **Мониторинг**: Следите за графиком онлайна, статистикой запросов, затратами на AI и живым чатом прямо в интерфейсе. Ошибочные аккаунты можно выгрузить кнопкой **Export CSV**.

## 🏗 Архитектура / Структура проекта

```text
src/
├── accounts/      # Парсинг, валидация и управление списком аккаунтов
├── bots/          # Логика tmi.js, пул соединений, индивидуальные контроллеры ботов
├── chat/          # Трекинг контекста, генерация сообщений, логирование чата
├── config/        # Загрузка, валидация и сохранение конфигурации
├── database/      # Инициализация SQLite, миграции, модели данных
├── services/      # Дополнительные сервисы (ViewerService на базе Puppeteer)
├── ui/            # React фронтенд, Zustand стор, Tailwind компоненты
├── utils/         # Утилиты: логгер (Winston), rate limiter, реконнектор, типы
├── main.ts        # Точка входа Electron (Main Process)
└── preload.ts     # IPC мост между Main и Renderer (UI)
```

- [ ] **Детекция активности стримера**: Анализ чата для идентификации сообщений стримера и подстройка поведения ботов.
