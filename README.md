# Анализ проекта Twitch Chat Viewer Bot

## Обзор
Это продвинутое приложение на Node.js, созданное с использованием Electron, React и ряда других технологий. Его основная функция — управление множеством аккаунтов ботов Twitch для просмотра каналов, симуляции активности зрителей и общения в чате с использованием как заранее заготовленных сообщений, так и ответов, генерируемых искусственным интеллектом (AI).

## Технологический стек
- **Бэкенд/Ядро:** Node.js, TypeScript
- **Десктопный фреймворк:** Electron (позволяет запускать веб-приложение как обычную программу на компьютере).
- **Фронтенд/Пользовательский интерфейс (UI):** React, Vite, Tailwind CSS, Zustand (для управления состоянием приложения на стороне клиента).
- **Интеграция с Twitch:**
  - `tmi.js` (популярная библиотека для работы с чатом Twitch).
  - `puppeteer` и `puppeteer-extra` (с плагином `stealth` для имитации реальных браузеров, просмотра трансляций и обхода антибот-систем Twitch).
- **База данных:** SQLite (библиотека `better-sqlite3`) — легкая локальная база данных для хранения истории сообщений и кэширования.
- **ИИ Интеграция:** OpenAI API (`openai`) для динамической генерации сообщений в чате на основе контекста.
- **Логирование:** Winston (с ежедневной ротацией файлов, чтобы логи не занимали слишком много места).
- **Визуализация данных:** Chart.js (для построения графиков активности в пользовательском интерфейсе).

## Архитектура проекта (директория `src/`)

Проект отлично структурирован и разбит на логические модули:

### 1. Ядро приложения (`src/main.ts`, `src/preload.ts`)
- **`main.ts`**: Главный процесс Electron. Это "сердце" программы. Он запускает приложение, инициализирует базу данных, менеджеры аккаунтов и ботов. Также он настраивает IPC (Inter-Process Communication) — каналы связи между скрытой серверной частью Node.js и видимым интерфейсом React, передавая статистику использования памяти, стоимость API ИИ и т.д.
- **`preload.ts`**: Безопасный "мост" между `main.ts` и интерфейсом React, определяющий, какие функции бэкенда может вызывать интерфейс.

### 2. Пользовательский интерфейс (UI) (`src/ui/`)
Написан на React.
- **Компоненты (`src/ui/components/`)**:
  - `Dashboard.tsx`: Главный экран, объединяющий остальные элементы.
  - `StatsBar.tsx`: Панель статистики (показывает онлайн ботов, скорость чата, расходы на ИИ, использование ОЗУ, аптайм).
  - `AccountsTable.tsx`: Таблица для управления загруженными аккаунтами.
  - `ControlsPanel.tsx`: Панель управления (старт/стоп ботов, настройки).
  - `OnlineChart.tsx`: График онлайна ботов (на базе Chart.js).
  - `ChatPreview.tsx`: Превью чата в реальном времени.
  - `SettingsModal.tsx`: Окно настроек.
- **Управление состоянием (`store.ts`)**: Использует Zustand для хранения данных, получаемых от бэкенда (статусы ботов, новые сообщения), чтобы UI мог мгновенно обновляться.

### 3. Менеджмент аккаунтов (`src/accounts/`)
- Отвечает за загрузку (из файлов или текста), парсинг, проверку (валидацию) и хранение данных аккаунтов ботов Twitch.

### 4. Управление ботами (`src/bots/`)
- **`IndividualBot.ts`**: Обертка для одного конкретного бота. Управляет его подключением к чату (через `tmi.js`), отправкой сообщений и обработкой отключений/банов.
- **`BotPool.ts`**: Контролирует массовое подключение ботов, создавая задержки, чтобы избежать блокировок со стороны Twitch за слишком частые запросы (rate limits).
- **`BotManager.ts`**: Главный дирижер. Содержит логику "плавающего" онлайна (периодическое отключение и подключение части ботов для имитации реальных людей), глобальные лимиты сообщений и специальные команды (например, массовые приветствия или эмоции `triggerGreeting`, `triggerEmotion`).

### 5. Логика чата и ИИ (`src/chat/`)
- **`ChatLogger.ts` / `ContextTracker.ts`**: Сохраняют историю сообщений (в БД и в памяти) для того, чтобы ИИ понимал контекст беседы.
- **`ConversationEngine.ts`**: Принимает решения о том, *когда* бот должен написать в чат, должен ли он ответить на упоминание или начать новую тему.
- **`MessageGenerator.ts` / `PersonaManager.ts`**: Отвечают за генерацию текста сообщений (через OpenAI или из заготовленных фраз) с учетом "характера" бота.

### 6. Сервисы (`src/services/`)
- **`AIService.ts`**: Взаимодействует с OpenAI (ChatGPT), отслеживает токены и стоимость запросов.
- **`ViewerService.ts`**: Управляет скрытыми окнами Puppeteer, которые "смотрят" стрим, создавая реальные просмотры и эмулируя активность (запросы к Twitch), а также собирает статистику потребления ОЗУ и трафика этими окнами.

### 7. База данных (`src/database/`)
- **`Database.ts`**: Локальная база SQLite для хранения логов чата и кэширования ответов ИИ (чтобы не платить OpenAI дважды за один и тот же ответ). Содержит систему миграций (настройка таблиц при первом запуске).

### 8. Конфигурация (`src/config/`)
- Хранит настройки (канал, ключи API, вероятности ответов, лимиты сообщений) с возможностью обновления через интерфейс.

### 9. Утилиты (`src/utils/`)
- Вспомогательные файлы: лимитеры запросов (`RateLimiter.ts`), логика переподключений (`Reconnector.ts`), типы данных TypeScript (`types.ts`) и логгер (`Logger.ts`).
